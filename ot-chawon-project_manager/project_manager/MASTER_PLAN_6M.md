# Ot-chawon 6개월 마스터 플랜 (PM)

## 1. 목표
- 로그인에서 시작해 상품 탐색, 주문, 3D 뷰, AI 피팅까지 이어지는 커머스 MVP 완성
- 출시 가능한 운영 품질(로그/모니터링/보안/QA 게이트) 확보

## 2. 기간/운영 단위
- 총 기간: 6개월
- 스프린트: 2주 단위 (총 12개 스프린트)
- 릴리즈 마일스톤: M1(기반), M3(커머스+3D), M5(AI 베타), M6(출시)

## 3. 월별 목표
### Month 1: Foundation + Auth
- 아키텍처/도메인/코딩 규칙 확정
- 로그인/세션/권한 기본 구현
- 기본 상품 조회 API 및 로그인 UI 완성

### Month 2: Commerce Core
- 상품 목록/상세/검색/필터
- 장바구니/주문 생성
- 상품 이벤트 로깅(조회/클릭/장바구니)

### Month 3: 3D Experience
- 상품 상세 3D 뷰어 연동
- 3D 이벤트 로그(회전/줌) 수집
- 3D 자산 QC 파이프라인 정착

### Month 4: AI Fitting Beta
- 사이즈 추천 고도화
- 피팅 입력 UX + 추천 근거(reason) 노출
- 오프라인 평가 지표/검증 체계 확립

### Month 5: Reliability + Security
- 성능/부하 테스트
- 관측성 강화(메트릭, 로그, 알림)
- 계정보안, 감사 추적, 이상 로그인 탐지 보강

### Month 6: Stabilization + Launch
- 통합 QA/UAT/회귀 테스트
- 릴리즈 리허설
- 운영 인수인계 문서/런북 완성

## 4. 릴리즈 게이트 (공통)
1. 기능 게이트: 요구사항 DoD 충족
2. 품질 게이트: 테스트/QA 체크리스트 통과
3. 운영 게이트: 모니터링/로그/알림 연결 완료
4. 보안 게이트: 인증/권한/민감정보 점검 완료

## 5. PM 운영 원칙
- PM 선계획 -> Sub Agent 실행 -> PM 통합 보고 순서 고정
- 스프린트 시작 전 WBS 확정, 종료 시 산출물/리스크/다음 액션 확정
- Sub Agent는 PM 배정 문서 외 작업 금지
