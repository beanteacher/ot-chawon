# WBS 상세 계획 (6개월)

## WBS-L1 Workstreams
1. PM/Product
2. Backend
3. Frontend
4. AI/ML
5. 3D Graphics
6. DevOps/SRE
7. QA
8. UI/UX

## WBS-L2 + 산출물 + 완료기준

### 1) PM/Product
- 요구사항 관리/우선순위/로드맵
- 산출물: PRD, 스프린트 계획서, 리스크 로그
- 완료기준: 매 스프린트 목표/범위/완료판정 명확

### 2) Backend
- Auth(로그인, 세션, 토큰, 로그인 시도 로그)
- Catalog(상품/브랜드/카테고리/옵션)
- Order(장바구니, 주문, 결제 상태)
- Tracking(상품 이벤트, 감사 로그)
- 산출물: OpenAPI v1~v3, schema-v1~v3, 서비스 구현 코드
- 완료기준: 주요 API 계약 테스트 통과 + 운영 로그 적재 확인

### 3) Frontend
- 로그인/회원 UI
- 상품 목록/상세/검색/필터
- 장바구니/주문 UI
- 운영 상태 대시보드
- 산출물: Next.js 화면/컴포넌트/상태 관리 코드
- 완료기준: 주요 사용자 여정(E2E) 통과

### 4) AI/ML
- 사이즈 추천 규칙->모델 고도화
- 피팅 입력 피처 설계
- 추천 근거(reason) 설명성 체계
- 산출물: API 스펙, 평가 리포트, 테스트
- 완료기준: 기준 정확도/안정성 도달

### 5) 3D Graphics
- 3D 자산 생산/정리/QC
- 웹 최적화(GLB, 텍스처)
- 3D 상호작용 이벤트 정의
- 산출물: 자산 매니페스트, QC 리포트, 연동 가이드
- 완료기준: 주요 카테고리 3D 적용률 목표 충족

### 6) DevOps/SRE
- 로컬/개발/스테이징 실행 표준
- CI/CD, 배포 자동화
- 모니터링/로그/알림
- 산출물: runbook, compose/k8s, 대시보드/알림 정책
- 완료기준: 배포 리허설 성공, 장애 대응 시간 목표 충족

### 7) QA
- 테스트 전략(Unit/Contract/E2E)
- 스모크/회귀 자동화
- 릴리즈 게이트 운영
- 산출물: 테스트 케이스, 자동화 스크립트, 리포트
- 완료기준: 릴리즈 블로커 0건

### 8) UI/UX
- 디자인 시스템(토큰, 컴포넌트)
- 핵심 화면(로그인/상품/주문/피팅) 설계
- FE handoff 기준 운영
- 산출물: Figma 프레임, 컴포넌트 명세, 상태별 문구
- 완료기준: FE 구현 가능 수준의 handoff 완료

## 의존성 맵
1. 로그인 기능: UI/UX -> FE -> BE -> QA
2. 상품 이벤트 분석: BE 스키마/API -> FE 이벤트 송신 -> AI/ML 집계 규칙 -> QA 검증
3. 3D 경험: 3D 자산 -> FE 뷰어 -> BE 이벤트 적재 -> PM KPI 보고
